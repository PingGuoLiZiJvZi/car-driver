# 目标
- 主控板：装载 Debian OS 的 Orange Pi Zero 3 (架构：ARM64)。
- 硬件架构：通过 8 个 GPIO 驱动两个迷你 L298N 模块（无 EN 使能引脚），分别控制 4 个 TT 马达构成的麦克纳姆轮底盘。
- 软件架构：使用 C 语言和 wiringOP 库实现底层控制，编译为动态链接库 (.so)，向上层（如 Python）暴露易用的运动控制接口。

# 核心要求

## 1. 引脚管理与迷你 L298N 驱动逻辑
- 必须使用一个长度为 8 的全局数组来保存 GPIO 引脚配置，数组索引严格映射为：[FL_IN1, FL_IN2, FR_IN1, FR_IN2, RL_IN1, RL_IN2, RR_IN1, RR_IN2]（FL=左前，FR=右前，RL=左后，RR=右后）。
- **PWM 驱动逻辑限制**：迷你 L298N 没有独立的 PWM 使能引脚，只能通过 IN1 和 IN2 控制。
  - 正转：IN1 输出 softPwm 信号，IN2 输出持续低电平 (LOW)。
  - 反转：IN1 输出持续低电平 (LOW)，IN2 输出 softPwm 信号。
  - 停止：IN1 和 IN2 均输出持续低电平 (LOW)。
- 必须提供初始化接口 `int chassis_init(const int pins[8])`，并在内部维护初始化状态。未初始化时拒绝执行任何运动函数。

## 2. 运动控制与逆运动学解算
- 提供全向移动接口：`int chassis_set_velocity(float vx, float vy, float omega)`。
- **坐标系与输入范围**：
  - `vx`：X轴平移，右为正，左为负。
  - `vy`：Y轴平移，前为正，后为负。
  - `omega`：Z轴自转，顺时针为正，逆时针为负。
  - 设定这三个参数为归一化输入，取值范围均为 [-1.0, 1.0]。
- **解算与限幅归一化**：
  - 由于未提供具体的轴距和轮距，C 库内部需使用基于比例的简化版麦克纳姆轮逆运动学公式进行矢量合成。
  - 必须对解算后的四个电机目标值进行**比例限幅归一化**：如果某个电机的计算速度绝对值超出了 PWM 最大值（设为 100），则必须按比例等比缩小所有电机的速度，确保运动矢量方向不变。

## 3. 跨平台 Mock 与兼容性实现 (关键)
- 开发环境为 Ubuntu 24.04 (x86_64)，实际运行环境为 Orange Pi (ARM64)。
- 必须在代码中使用条件编译宏（如 `#ifndef __aarch64__`）对 wiringOP 的核心函数（如 `wiringPiSetup`, `pinMode`, `softPwmCreate`, `softPwmWrite`, `digitalWrite`）进行 Mock（桩函数替代）。
- 在 x86 环境下，Mock 函数只需向控制台（标准输出）打印对应的引脚操作日志即可；在 ARM64 环境下，则调用真实的 wiringOP 库。确保代码能在 Ubuntu 下顺利编译并验证解算逻辑。

## 4. 鲁棒性、安全性与日志
- 提供基于日志级别（DEBUG, INFO, ERROR）的打印功能，所有日志以英文形式输出。
- 提供 `chassis_cleanup()` 接口，用于在程序退出时安全拉低所有引脚电平并停止 PWM，防止电机失控。
- 内部状态校验必须严谨，防止段错误、数组越界或未初始化调用。

## 5. 构建系统配置
- 请同时提供一份 `Makefile`。
- Makefile 必须包含编译为动态链接库的目标规则，使用 `-fPIC` 和 `-shared` 选项生成 `libchassis.so`。
- 需处理好 wiringPi 库的链接参数（如 `-lwiringPi -lpthread`），并支持通过环境变量或参数切换跨平台 Mock 编译模式。
